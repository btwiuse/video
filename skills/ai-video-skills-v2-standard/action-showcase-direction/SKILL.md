# Action Showcase Direction

## Core Role

Design a pure action showcase as a progression of states. A showcase is not a fight missing a story; it proves how a weapon, body, armor, power, or movement system behaves.

## Showcase Questions

```text
what is being showcased?
what makes it different?
what are the 3-5 states of escalation?
what is the strongest proof shot?
what final icon should remain?
```

## State Progression

Use this progression for most showcase clips:

```text
idle identity -> activation -> first function proof -> escalation / variation -> strongest peak -> controlled recovery -> final icon
```

## Showcase Types

### Weapon showcase

Prove grip, balance, range, speed, weight, impact, recovery, and final pose.

### Armor / mechanical showcase

Prove deployment, lock, joint motion, weight shift, actuator force, protection, recoil, and shutdown.

### Power / magic showcase

Prove source, buildup, shape, rule, target, consequence, and residue.

### Movement-system showcase

Prove start, acceleration, obstacle handling, direction change, braking, and final stance.

### Character combat-language showcase

Prove attitude, preferred distance, signature rhythm, strength level, and silhouette.

## 15-Second Showcase Template

```text
0-2s: clean hero introduction, readable silhouette, weapon/power dormant
2-4s: activation detail, hand, eye, mechanism, energy source, or stance change
4-7s: first function proof, one clear action with visible result
7-10s: escalation, second angle or movement variation
10-13s: strongest peak, full-body and environment consequence
13-15s: recovery, final pose, residue fades, hold the icon
```

## 30-Second Showcase Template

```text
0-4s: identity and system introduction
4-8s: activation and first function
8-13s: handling / mobility proof
13-18s: obstacle or resistance proof
18-24s: strongest peak / full capability
24-30s: aftermath, shutdown, final hero frame
```

## Shot Design Rules

- Use close-up for activation, grip, material, eye, mechanism, or power source.
- Use medium/wide for full-body action and weapon path.
- Use extreme wide only when scale or environment consequence matters.
- Vary shot size by function, not randomly.
- Do not repeat the same swing, dash, pose, or explosion.
- If no opponent exists, use targets, obstacles, environment, training dummy, falling debris, or spatial marks to prove effect.


## Final Prompt Pattern

```text
Duration 15 seconds. A pure action showcase of [subject]. Start with [identity image]. The system activates through [visible trigger]. Show three escalating function proofs: [first], [second], [strongest]. Each action has clear body support, readable motion path, visible consequence, and controlled recovery. End on [final icon] and hold for one second.
```

## Common Failures

- **Looks like random fighting**: define the showcased system and state progression.
- **Too repetitive**: each beat must prove a different function.
- **No weight**: add support, recoil, braking.
- **No final image**: design a held hero pose.
- **Effects detached from body**: connect power to hand, weapon, core, breath, machine, or environment.

## Negative Constraints

```text
no repetitive moves, no random VFX, no unclear weapon identity, no idle middle section, no disconnected effects, no camera hiding the function, no final pose missing, no action without visible consequence
```
